Cuju Quick Start Guide

The environment prepare

  • Assume you have already builded 3 nodes (nfs, primary and backup node).
    • A proper topology like this
  • Open the Intel virtualization support (VT-x) in your bios.
  • Install OS: Ubuntu 16.04 LTS (Linux kernel 4.4.48)
  • Change your current Linux config
    • Download Linux-4.4.48 source
     $ cd linux-4.4.48
 $ cp /boot/config-'uname -r' ./.config
 $ make menuconfig

 Disable following item:
   Processor type and features
     [ ] x86 PAT support
   Kernel hacking
     [ ] Filter access to /dev/mem
  • Install the NFS service (Network FileSystem) in nfs node, and create a NFS forder placing the VM image.
  • Primary and backup node mount the NFS forder from nfs node.
    • In nfs node, go to your nfs forder, then download Cuju and build a VM image file (Ubuntu-16.04 in our case), they will be synced with primary and backup node.

Cuju Install

  • In primary node, go to your Cuju path
    •  $ ./configure --disable-pie --enable-kvm --enable-cuju
   --target-list=x86_64-softmmu --enable-trace-backend=simple 
 $ make clean
 $ make
 $ cd cuju-kvm-mod
 $ ./configure
 $ make clean
 $ make

Cuju Testing Steps

  1. Download necessary scripts: ft-scripts.tar.gz
    • Uncompress to your work path
    • Seven scripts in it: pre.sh, reinsmodkvm.sh, set-net2.sh, set-vmft.sh, ftmode.sh, run_ft and recv
  2. Before you launch VM in any node, run pre.sh both in primary and backup node.
  3. execute ./run_ft in primary node to boot the VM
  4. execute ./recv in backup node to boot the backup VM
  5. execute ./ftmode.sh in primary to enter FT mode
  6. Remote to VM and ping 8.8.8.8 or any online server
  7. Kill the VM in primary node
  8. Check if the connection of VM and its ping action is still alive or not
