Cuju Quick Start Guide

  • All 3 nodes install Ubuntu-12.04.1
  • Add no password in node 1&2
  • $ sudo apt-get update
  • $ sudo apt-get install vim gcc make gdb fakeroot build-essential kernel-package libncurses5 libncurses5-dev zlib1g-dev libglib2.0-dev qemu xorg bridge-utils openvpn vncviewer
  • Install Linux Kernel 3.5.0(disable following items in menuconfig):
    • [ ]X86 PAT support (Processor type and features)
    • [ ]Filter access to /dev/mem (Kernel hacking)
  • Set up iptable and network forwarding to allow all 3 nodes connecting to Internet.
  • Set up NFS.
  • Edit /etc/qemu-ifup & /etc/qemu-ifdown in both node 1&2 as follow to avoid using it :
    •  #!/bin/sh
 exit
 ...
  • Compile CUJU on all 3 nodes:
    • QEMU
      • $ cd $PATH_OF_YOUR_CUJU/
      • $ ./configure --disable-pie --enable-kvm
      • $ make clean
      • $ make -j8/
    • KVM
      • $ cd $PATH_OF_YOUR_CUJU/kvm
      • $ ./configure
      • $ make clean
      • $ make -j8/

Testing Steps :
  1. Before you launch VM in any node, run pre.sh
  2. execute ./run_ft in node1 to boot the VM.
  3. execute boot.sh or vncviewer: 5900 in node1 to fix VM stuck at booting stage.
  4. execute ./recv in node2 to boot the backup VM.
  5. execute ./ftmode.sh in node1 to enter FT mode.
  6. enter the VM in node1 and ping 8.8.8.8
  7. Kill the VM in node1.
  8. execute vncviewer:5900 in node2 to check if the ping action is still alive or not.
  • You can find the mentioned above scripts in the download section.
